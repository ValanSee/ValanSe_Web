## 🎯 화면 목적

비로그인 사용자에게 서비스 소개와 카카오 로그인 진입점을 제공한다.

## 🚪 진입 경로

- 토큰 없음/만료된 상태에서 `/` 진입 시 리다이렉트
- 로그아웃 직후
- **로그인 필요 기능(마이/투표/댓글 등)에서 미인증 시** `redirect` 파라미터와 함께 리다이렉트 (`/entry?redirect=<원래경로>`)
- 외부에서 진입 (TODO: 딥링크 정책 확인)

## 📐 레이아웃 구성

1. **상단 헤더** — 뒤로가기 버튼(`Header showBackButton`). 막다른 길 방지용
2. **로고/일러스트** (TODO: 디자인 확인)
3. **서비스 소개 카피**
4. **카카오 로그인 버튼** (KakaoLoginButton)
5. **로그인 없이 둘러보기** 링크 → `/main`

## 🧩 섹션별 상세 명세

### 1. KakaoLoginButton

**사용자 액션**

- 클릭 → 카카오 OAuth 인가 페이지 이동
- 인가 후 `/oauth/kakao/redirect`로 콜백

### 2. 뒤로가기 버튼

**사용자 액션**

- 클릭 → `/main`(홈)으로 이동
- 보호 페이지(`/my` 등)로 `router.back()` 하면 가드가 다시 `/entry`로 보내 루프가 발생하므로, 공개 홈으로 명시적으로 탈출시킨다.

## 🔄 사용자 시나리오

**시나리오 1: 최초 로그인**

1. `/entry` 진입
2. 카카오 로그인 버튼 클릭
3. 카카오 인가 → 콜백 → 신규면 `/onboarding`, 기존이면 `/main`

**시나리오 2: 로그인 필요 기능에서 튕겨온 뒤 로그인하지 않음**

1. 마이/투표 등에서 미인증 → `/entry?redirect=<원래경로>` 진입
2. 뒤로가기 버튼 클릭 → `/main`(홈)으로 복귀 (막다른 길 없음)
