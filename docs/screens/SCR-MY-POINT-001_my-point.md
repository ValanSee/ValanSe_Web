## 🧩 섹션별 상세 명세

### 1. 포인트 헤더 섹션

**목적**: 현재 보유 잔액을 강조 노출하고 칭호 상점으로의 진입을 제공.

**표시 데이터**

| 항목 | 출처 | 비고 |
|---|---|---|
| 현재 잔액 | `GET /member/point-history` 응답의 `pointHistory[0].remainingPoint` | 가장 최신 항목의 잔액 = 현재 잔액. 내역이 없으면 0. |

**사용자 액션**

- "칭호 상점 가기" 버튼 클릭 → `/my/titles` 이동 (TODO: SCR-MY-TITLE-001 작성 예정)

**상태별 처리**

- 잔액 없음: `0P` 표시

### 2. 포인트 적립 정책 공지

**목적**: 포인트 적립 기준을 사용자에게 안내. 기본 접힘 상태.

**표시 데이터** (정적)

| 항목 | 적립량 |
|---|---|
| 회원가입 | 40P |
| 게시물 작성 | 5P |
| 좋아요 받기 | 1P / 1개 |
| 핫이슈 선정 | 50P |
| 댓글 작성 | 1P (일일 3회 제한) |

**사용자 액션**

- 헤더 클릭 → 펼침/접힘 토글

### 3. 포인트 내역 리스트

**목적**: 적립·사용 내역을 시간 순으로 확인.

**표시 데이터**

| 항목 | 출처 | 비고 |
|---|---|---|
| 내역 설명 | `typeDescription` | 예: "댓글 작성", "핫이슈", "칭호 구매" |
| 일시 | `createdAt` | `YYYY.MM.DD HH:mm` 포맷으로 변환 |
| 변동량 | `amount` | 양수: `+50P` 파란색(`#4D7298`), 음수: `-300P` 회색(`#8E8E8E`) |

**사용자 액션**

- 없음 (조회 전용)

**상태별 처리**

- **로딩**: 페이지 단위 Loading 컴포넌트
- **에러**: thunk 실패 시 인증 만료로 간주 → `/entry`로 redirect (redirect URL 포함)
- **빈 상태**: "아직 받은 포인트가 없어요"

## 🔄 사용자 시나리오

**시나리오 1: 잔액·내역 확인**

1. `/my`에서 "내 포인트" 영역 클릭
2. `/my/point` 진입 → 잔액 + 내역 표시
3. 정책 공지 펼쳐서 적립 기준 확인

**시나리오 2: 칭호 상점 진입**

1. `/my/point`에서 "칭호 상점 가기" 클릭
2. `/my/titles` 이동 (후속 작업 SCR-MY-TITLE-001)

## ✅ 수용 기준 (Acceptance Criteria)

- Given 로그인 상태
  When `/my/point` 진입
  Then 가장 최신 내역의 `remainingPoint`가 잔액으로 표시됨

- Given 내역이 비어있음
  When `/my/point` 진입
  Then 잔액은 `0P`, 리스트 영역은 "아직 받은 포인트가 없어요" 표시

- Given 적립 내역에 음수(`amount < 0`)가 포함됨
  When 리스트 렌더링
  Then 해당 항목의 변동량은 회색으로, `-300P` 형태로 표시
