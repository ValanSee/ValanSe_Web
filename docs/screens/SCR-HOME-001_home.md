## 🎯 화면 목적

사용자가 앱 진입 시 가장 먼저 보는 메인 화면. 오늘의 핫이슈 밸런스 게임에 빠르게 참여하도록 유도하고, 카테고리별 게임 탐색과 직접 게임 생성으로의 진입점을 제공한다.

## 🚪 진입 경로

- 앱 최초 실행 시 (로그인 → 홈)
- 하단 네비게이션 바의 "홈" 탭 클릭

## 📐 레이아웃 구성

화면은 위에서부터 다음 4개 섹션으로 구성된다.

1. **상단 핫이슈 카드** 클릭 시 핫이슈 탭으로 이동
2. **밸런스 게임 만들기 CTA 카드**
3. **카테고리 바로가기 그리드** (3열)
4. **하단 네비게이션 바** (5개 탭)

## 🧩 섹션별 상세 명세

### 1. 핫이슈 섹션

**목적**: 24시간 한정으로 진행되는 대표 밸런스 게임을 노출하여 참여를 유도

**표시 데이터**

| 항목 | 출처 | 비고 |
|---|---|---|
| 핫이슈 타이틀 ("오늘의 핫이슈") | 정적 텍스트 | - |
| 총 참여자 수 ("1,234명 참여") | `GET /votes/best` api 의 `totalParticipants` | 1,000 단위 콤마 |
| 선택지 A 텍스트 | 동일 API의 `options[0]` | - |
| 선택지 B 텍스트 | 동일 API의 `options[1]` | - |

**사용자 액션**

- 핫이슈 카드 클릭 → 핫이슈 탭으로 이동

**상태별 처리**

- **로딩**: 스켈레톤 UI (회색 박스)
- **에러**: "핫이슈를 불러올 수 없어요" 메시지 + 재시도 버튼
- **빈 상태**: 진행 중인 핫이슈가 없을 경우 -> 미정

### 2. 밸런스 게임 만들기 CTA

**목적**: 사용자가 직접 게임을 만들도록 유도

**표시 데이터**

- 정적 텍스트 "밸런스 게임 만들기"
- 우측 펜 아이콘

**사용자 액션**

- 카드 전체 영역 탭 → 게임 생성 화면(`/create`)으로 이동

### 3. 카테고리 그리드

**목적**: 사용자가 관심 카테고리로 빠르게 진입

**표시 데이터**

| 카테고리 | 아이콘 | 라우팅 경로 |
|---|---|---|
| 음식 | 포크/나이프 | `/balanse?category=FOOD` |
| 연애 | 하트 | `/balanse?category=LOVE` |
| 기타 | 별 | `/balanse?category=ETC` |

**사용자 액션**

- 카테고리 카드 탭 → 해당 카테고리 게임 목록 화면으로 이동

### 4. 하단 네비게이션 바

**목적**: 주요 화면 간 이동

| 탭 | 아이콘 | 경로 | 활성 조건 |
|---|---|---|---|
| 홈 | 집 | `/main` | 현재 화면 |
| 인기 | 불꽃 | `/poll/hot` | - |
| 밸런스 | 저울 | `/balanse` | - |
| 만들기 | 펜 | `/create` | 로그인 필요 |
| 내 정보 | 사람 | `/my` | 로그인 필요 |

**비고**: 현재 화면에서는 "홈" 탭이 활성 상태 (파란색 강조)
